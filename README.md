## Part 1 — Mobile Web App Setup

### Step 1: Create your Firebase project

1. Go to https://console.firebase.google.com
2. Click **Add project** → name it (e.g. `walk-pipeline`) → Continue
3. Disable Google Analytics (not needed) → **Create project**
4. In the left sidebar, click **Firestore Database** → **Create database**
   - Choose **Start in test mode** (allows reads/writes without authentication)
   - Select a region close to you (e.g. `europe-west2` for UK)
5. Go to **Project settings** (gear icon) → **Your apps** → click the web icon `</>`
6. Register the app with a nickname (e.g. `walk-app`)
7. Copy the `firebaseConfig` object shown — you will need it next

### Step 2: Add your Firebase credentials to the app

Open `app/index.html` and find this block near the bottom:

```javascript
const FIREBASE_CONFIG = {
  apiKey:            "REPLACE_WITH_YOUR_API_KEY",
  authDomain:        "REPLACE_WITH_YOUR_PROJECT_ID.firebaseapp.com",
  projectId:         "REPLACE_WITH_YOUR_PROJECT_ID",
  storageBucket:     "REPLACE_WITH_YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "REPLACE_WITH_YOUR_MESSAGING_SENDER_ID",
  appId:             "REPLACE_WITH_YOUR_APP_ID"
};
```

Replace each `REPLACE_WITH_...` value with the values from your Firebase console.

### Step 3: Add the app to your iPhone home screen

**Recommended: serve locally over Wi-Fi**
```bash
cd app
python3 -m http.server 8080
```
On your iPhone, open Safari and go to `http://YOUR_COMPUTER_IP:8080`
(find your IP with `ifconfig | grep "inet "` on Mac/Linux).

Then tap **Share → Add to Home Screen → Add**.

### How to use the app during a walk

1. Open the app → fill in the pre-walk questionnaire → tap **START WALK**
2. Walk — the GPS records a point every 10 seconds, shown on the live map
3. Tap **STOP WALK** → fill in the post-walk questionnaire → tap **SUBMIT**
4. Data saves to Firebase automatically; a local backup is also kept in the phone's browser storage

## Part 2 — Python Data Extraction

### Install dependencies

```bash
pip install firebase-admin pandas
```

### Get your Firebase service account key

1. Firebase console → **Project settings** → **Service accounts**
2. Click **Generate new private key** → download the JSON file
3. Save it as `scripts/firebase_key.json`
4. This file is in `.gitignore` — never commit it to GitHub

### Export Apple Health data

1. Install the **Health Auto Export** app on your iPhone (free tier is sufficient)
2. Configure it to export: Heart Rate, Oxygen Saturation, Step Count, Active Energy Burned
3. After one or more walks, export as **JSON** and save to your computer

### Run the script

```bash
python scripts/extract_health_data.py \
    --firebase-key scripts/firebase_key.json \
    --health-export ~/Downloads/health_export.json \
    --output-dir data
```

Outputs:
- `data/sessions.csv` — one row per walk, opens in Excel or Numbers
- `data/sessions.json` — what the dashboard reads (includes GPS paths for maps)

### Push the updated data to GitHub

```bash
git add data/sessions.json
git commit -m "Update walk session data"
git push
```

## Part 3 — Dashboard (GitHub Pages)

The dashboard at `dashboard/index.html` is fully built and ready. It reads
`data/sessions.json` and renders the data table, route maps, and charts automatically.

### To publish when you have real data

1. Make the repository **public** (Settings → Danger Zone → Change repository visibility)
2. Go to **Settings → Pages → Source** → select **GitHub Actions**
3. Push any change to trigger deployment, or run the workflow manually from the Actions tab
4. Dashboard live at: `https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/`

Until you do those steps, nothing is publicly visible.

## Security notes

- `scripts/firebase_key.json` must be in `.gitignore` and never committed
- Tighten Firestore security rules before going public (see below)
- Review `data/sessions.json` before publishing — ensure it contains only your own data

### Recommended Firestore rules before going public

In Firebase console → Firestore → Rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /walk_sessions/{sessionId} {
      allow write: if request.time < timestamp.date(2027, 1, 1);
      allow read: if false;
    }
  }
}
```
